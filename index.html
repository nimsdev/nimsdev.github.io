Website Redesign Project

I helped Chase with his online presence. The old website was dated, and it needed a makeover. Chase gave me some ideas, I added mine, and a new site was created!

I'll start with the old version of the site, and show what I changed, and why I made those changes.


Old Header


Chase's company logo is large, but the old developers shrunk it to fit it in the nav bar. It doesn't look awful at all, but it's tough to make out some of the letters in the logo, in this case it was Chase's name. The main problem with the old header is the video. There was no reason at all for that to be included in the header. That shows up on every page. I have no clue why the previous developers did that, but removing it was the first thing I did.

Landing Page

I added a landing page to the site. The page gave users a brief outline on the services Chase is able to provide, testimonials, and an email newsletter sign up.


Content Pages

The old pages can be seen below. The main content pages were in need of a new, modern design. The pages in the previous version weren't really visually appealing. The fonts, the background images, and the alignment needed to go.


My Content Pages

I made completely new pages detailing the services offered by the company. Since the logo for Chase Jackson Peak Performance was shaped like a box (image dimensions), the navigation took up some space. To accommodate this, the header went away when users scrolled down, and came back when they scrolled up. The new navigation bar can be seen on the screenshot of one of the new pages I created.



Blog

I also added a blog to the site. There was a blog page before, but it wasn't really a blog. The old developers basically asked for the blog content, and put it into a bootstrap modal. Users couldn't comments, but most importantly they couldn't be edited or deleted by the owner of the site unless he had web dev knowledge.

I developed a new blog section using WordPress. Now Chase can edit blogs posts, create new ones, and have his site members comment on the articles he writes.

Crumyum

Hybrid Mobile App Demo

  • Sketch
  • HTML/CSS
  • Balsamiq
  • Ionic
  • Angular


Crumyum is an app that gives you a list of options (things to eat, types of foods, etc.), and then recommends something for you to eat. The end of this process brings users to a page a meal page. On this page, there is a description of the meal, ingredients, and instructions on how to make the meal.

The app is meant to be simple; easy to use, and sort of self explanatory. The main page of the app gives you five main categories to start. From there you will eventually land on a meal page that falls under one of the five main categories.

Crumyum can essentially be used by everyone that wants to learn how to cook, or just wants a meal idea when they're hungry, or getting ready to prepare something to eat.



Option pages are what the user sees before they reach a meal page. Depending on what users select, the can see a varieyu of pages before a meal is recommended. Since we wanted to make the app really simple, the entire process usually takes less than a minute.




The meal page (results) shows a recommended meal based on the options a user chose. The user will then see the meal they landed on, and get some information about it. Users will have all ingredients and instructions allowing them to make the meal if they choose to. Depending on the meal, users will get some addition information as well. Additional information can vary from: prep & cook time, estimated calorie count, and/or serving size. If the meal doesn't suit them, they can simply start the process over, and hopefully find something to eat!

Schoolitup

Photo Sharing App | UX

  • Sketch
  • HTML/CSS
  • Laravel Framework
  • PHP
  • Bootstrap
  • Photoshop


Schoolitup was a photo sharing app used by students. Users upload anything that relates to their school (sports, campus life, etc.). The site gives its users an opportunity to see life at other schools from a student's perspective.

Redesigning: The site was not very user friendly to begin with. When users visited the url (schoolitup.com), they would see a page with an image grid containing 90 photos. There were three different thumbnail sizes: small 45*45px, medium 90*90px, and large 145*145px. There weren't any descriptions, image tags, or user information relating to the image.


Old Main Page Layout


Actual grid had 90 photos changing in size depending on how often images were viewed.


New Main Page Layout




First Look

My first task was to create a landing page. Schoolitup was a new application, and nobody knew what it was used for. This page included a brief description explaining what the site was used for, a list of schools actively using the app, a few user profile links to top users, and a registration form at the bottom.



To make the site more interactive, we added more features (makes sense). After adding new developers to the team, we gave users actual profiles, the ability to follow one another, and the action to like photos. Users also had a photo feed from the people they followed, and received notifications if their image was liked or commented on, or if they had a new follower.


The user profile page showed photos uploaded by users with two images per row (compared to three on the main image pages). The third column was for user profile boxes (which were fixed). Profile boxes showed the profile picture of the user you were visiting, their stats (followers, following & image count), their name & username, their school, and a follow button.


Image cards with user profile picture, name, username (linking to profile), main image, number of likes the photo has, users that like it, image description, and image tags. A 3*4 row of these cards replaced the previous image grid. It gave the site a more organized, and cleaner look.
The last major upgrade was the upload system. In order to keep photos organized by school, we made each registered school a category. If this wasn't done, users could type their school name, and images would have been scattered. Example: writing Ottawa U / University of Ottawa would result in different school pages. This could have caused school pages to merge photos from different schools as well. We also added required fields to give the photos some character. Users had to make a title, write a description, select their school, and add related image tags.
